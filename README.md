# 7月投顾与T0奖励惩罚核算工具

这是水清南路营业部内部使用的财务辅助网页工具，用于核算 2026 年 7 月投顾与 T0 相关奖励、手工更正、剥离客户和 Excel 台账导出。

## 文件说明

- `index.html`：完整网页程序，已内置页面、样式、计算逻辑和 Excel 导出逻辑。
- `server.cjs`：本地预览用的小服务文件，GitHub Pages 部署时不是必需文件。
- `README.md`：本说明文件。

## 使用方式

1. 直接打开 `index.html`，或上传到 GitHub Pages 后访问网页。
2. 在页面中选择员工，填写投顾客户、T0客户、订单、撤单、创收和交易量。
3. 系统会自动刷新营业部奖励、上海分公司奖励、得分、剥离客户和计算过程。
4. 如有正式统计表确认的排名奖、联络员奖、扣回或额外奖励，使用“手工更正”录入。
5. 点击顶部“以 Excel 形式导出台账”导出核算结果。

## 数据存储

本工具不需要登录，也不连接服务器数据库。录入数据保存在当前浏览器本地存储中。

注意：

- 换浏览器或换电脑后，本地数据不会自动同步。
- 清空浏览器缓存或点击页面中的“清空全部数据”会删除本地台账数据。
- 上传到 GitHub Pages 只负责展示网页，不会保存或接收你的业务数据。

## 当前规则口径

当前版本重点适配 2026 年 7 月：

- 上海分公司投顾奖励按 7 月新规则核算。
- 营业部投顾、营业部 T0 和上海分公司 T0 暂沿用已确认口径。
- 排名类、联络员、下午茶等依赖正式统计表或跨营业部排名的数据，作为“手工更正”录入，不自动判断。

## GitHub Pages 部署

最简单方式：

1. 新建一个 GitHub 仓库。
2. 把 `index.html` 和 `README.md` 上传到仓库根目录。
3. 进入仓库 `Settings` -> `Pages`。
4. Source 选择 `Deploy from a branch`。
5. Branch 选择 `main`，目录选择 `/root`。
6. 保存后等待 GitHub Pages 生成访问链接。

如果打开后显示 404，请确认仓库根目录中确实存在小写文件名 `index.html`。
